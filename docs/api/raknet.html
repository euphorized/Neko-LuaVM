raknet

Wire-level packet hooks. Inspect, mutate, drop, or replace traffic before the host transport sees it.

module

Hooks

Send hooks fire just before each outgoing packet is handed to the transport. Recv hooks fire just after each incoming packet is parsed and before the game observes it.

raknet.add_send_hook(callback: function(Packet)) → () raknet.add_recv_hook(callback: function(Packet)) → () raknet.clear_send_hooks() → () raknet.clear_recv_hooks() → ()

Multiple hooks can be registered. They run in registration order, each receiving the same Packet userdata. Mutations made by one hook are visible to the next.

Packet

The userdata passed to hook callbacks. Lives only for the duration of the call.

packet.PacketId → number packet.Buffer → buffer packet.AsBuffer → buffer packet.Size → number packet.Length → number packet:SetData(buf: buffer) → () packet:SetPacketId(id: number) → ()

Buffer and AsBuffer are aliases. Size and Length are aliases. Reading Buffer returns a fresh copy each time, mutating that copy does not affect the in-flight packet, call SetData to commit changes back.

Example

lua 
1-- log every outgoing packet
2raknet.add_send_hook(function(p)
3    print(string.format("send id=%d size=%d", p.PacketId, p.Size))
4end)
5
6-- rewrite the first byte of one specific opcode
7raknet.add_send_hook(function(p)
8    if p.PacketId ~= 0x83 then return end
9    local b = p.Buffer
10    buffer.writeu8(b, 0, 0xFF)
11    p:SetData(b)
12end)

Lifecycle

Hooks survive script reruns until clear_send_hooks or clear_recv_hooks is called, or the VM is reinitialised. Be deliberate about deregistration, an orphaned hook can starve the dispatcher and stall the game's network thread under heavy traffic.
