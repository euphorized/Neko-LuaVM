gctab

Locate live host tables that contain a given string key, then mutate the matching value slots in place. The lookup walks the host's interned string table once and caches addresses across calls.

module stable

Overview

The library exposes four functions and one opaque cache shape. The cache is a Lua table, but you should treat it as private — the structure may change between versions.

gctab.find(keys) → cache gctab.set(cache, key, value) → number gctab.set(cache, map) → number gctab.refresh(cache) → number gctab.dump(cache) → ()

find

Walks the host's GC and resolves each requested string to its interned TString*. For every match, scans every live table on the heap for nodes keyed by that string, recording (table, node) tuples.

gctab.find(keys: { string }) → cache
Parameter Type Description
keys { string } An array of key names to search for. Order is irrelevant.

Returns

An opaque cache table with two fields:

Field Type Description
strings map Resolved interned string addresses, keyed by name. Missing keys are absent.
hits map For each key, an array of { table, node, string } hex addresses.

Example

find.lua 
1local cache = gctab.find({ "WalkSpeed", "JumpHeight" })
2
3for name, addr in pairs(cache.strings) do
4    print(name, addr, #cache.hits[name], "hits")
5end

set

Writes a new value into every node tracked by the cache for the given key. Two forms — single key/value, or a map of multiple keys to values.

gctab.set(cache: cache, key: string, value: any) → number gctab.set(cache: cache, map: { [string]: any }) → number

Writes are type-checked against the existing slot — only number and boolean slots accept rewrites. Anything else is skipped silently. The return value reports how many nodes were actually written.

Parameter Type Description
cache cache A cache produced by find.
key / map string / table Either a single key name plus a value, or a map of names to values.

Returns

Number of slots successfully written. Stale entries (slots whose key string no longer matches the cached pointer) are skipped without counting.

Example

set.lua 
1local cache = gctab.find({ "WalkSpeed", "JumpHeight" })
2
3local n = gctab.set(cache, {
4    WalkSpeed  = 100,
5    JumpHeight = 200,
6})
7
8gctab.set(cache, "WalkSpeed", 75)

refresh

Re-walks the heap and rebuilds the cache in place. Use after a server transition, character respawn, or any event that may have invalidated previously located tables.

gctab.refresh(cache: cache) → number

Returns the new total number of (table, node) hits. Strings that can no longer be resolved are dropped from the cache.

dump

Prints a human-readable summary of the cache to the console. For diagnostics only — do not parse the output.

gctab.dump(cache: cache) → ()

Notes

Scope. Many tables across the heap may share a key name — gun configs, UI state, server-mirror copies. set writes to every match. If you need finer control, iterate cache.hits[name] manually and use setgc on individual nodes.

Cost. A find call walks every page and every node of every table in the heap. On a typical session that's ~10k tables × ~16 nodes — fast enough for occasional rebuilds but not for every frame.

Stale safety. Each set revalidates the node's key tag and pointer before writing. A node that was a WalkSpeed slot at find time and is no longer one at set time is skipped without falling through to the wrong type.
